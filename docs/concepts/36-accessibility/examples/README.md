# Accessibility Examples

This directory contains practical code examples demonstrating accessibility best practices in Next.js.

## Examples

### 1. Semantic HTML (`01-semantic-html.tsx`)
Demonstrates semantic HTML:
- Proper HTML elements
- Navigation structure
- Form accessibility

### 2. ARIA Attributes (`02-aria-attributes.tsx`)
Shows ARIA attributes:
- ARIA labels
- ARIA roles
- ARIA live regions

### 3. Keyboard Navigation (`03-keyboard-navigation.tsx`)
Demonstrates keyboard accessibility:
- Focus management
- Keyboard traps
- Keyboard shortcuts

## How to Use

1. **Copy example code**: Adapt to your components
2. **Test with screen readers**: Verify accessibility
3. **Test keyboard navigation**: Ensure all features work
4. **Check color contrast**: Use tools to verify

## Key Concepts

### Semantic HTML
- Use proper HTML elements
- Provide structure and meaning
- Better for screen readers

### ARIA Attributes
- Enhance accessibility
- Provide context
- Describe dynamic content

### Keyboard Navigation
- All features keyboard accessible
- Logical tab order
- Focus management

## Best Practices

1. **Use semantic HTML**: Proper elements
2. **Add ARIA when needed**: Enhance clarity
3. **Test keyboard navigation**: Verify functionality
4. **Test with screen readers**: Real-world testing
5. **Check contrast**: WCAG compliance

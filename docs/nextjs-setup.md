# Next.js Project Setup Guide

A comprehensive step-by-step guide for creating a Next.js project with App Router.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js**: Version 18.17 or later
  - Check your version: `node --version`
  - Download from: [nodejs.org](https://nodejs.org/)
- **npm** or **yarn** or **pnpm** (comes with Node.js)
  - Check npm version: `npm --version`

## Method 1: Using create-next-app (Recommended)

The easiest way to create a Next.js project is using the official `create-next-app` CLI tool.

### Step 1: Create a New Project

Open your terminal and run one of the following commands:

```bash
# Using npm
npx create-next-app@latest my-nextjs-app

# Using yarn
yarn create next-app my-nextjs-app

# Using pnpm
pnpm create next-app my-nextjs-app
```

### Step 2: Configure Your Project

The CLI will prompt you with several configuration options. Here's what each option means:

#### **Would you like to use TypeScript?**
- **Yes** (Recommended): Adds TypeScript support for type safety
- **No**: Uses JavaScript instead

#### **Would you like to use ESLint?**
- **Yes** (Recommended): Adds ESLint for code linting
- **No**: Skips ESLint setup

#### **Would you like to use Tailwind CSS?**
- **Yes** (Recommended): Adds Tailwind CSS for styling
- **No**: You'll need to set up your own CSS solution

#### **Would you like to use `src/` directory?**
- **Yes**: Organizes code in a `src/` folder
- **No** (Recommended): Keeps files in the root directory (simpler structure)

#### **Would you like to use App Router? (recommended)**
- **Yes** (Recommended): Uses the modern App Router (Next.js 13+)
- **No**: Uses the Pages Router (legacy)

#### **Would you like to customize the default import alias?**
- **Yes**: Allows you to set a custom import alias (e.g., `@/components`)
- **No** (Recommended): Uses the default `@/*` alias

### Step 3: Navigate to Your Project

```bash
cd my-nextjs-app
```

### Step 4: Install Dependencies

If dependencies weren't installed automatically:

```bash
npm install
# or
yarn install
# or
pnpm install
```

### Step 5: Run the Development Server

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser to see your Next.js application.

## Method 2: Manual Setup

If you prefer to set up Next.js manually or need more control over the configuration:

### Step 1: Initialize a Node.js Project

```bash
mkdir my-nextjs-app
cd my-nextjs-app
npm init -y
```

### Step 2: Install Next.js and React

```bash
npm install next react react-dom
npm install -D typescript @types/node @types/react @types/react-dom
```

### Step 3: Create Project Structure

Create the following directory structure:

```
my-nextjs-app/
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── globals.css
├── public/
├── next.config.js
├── tsconfig.json
├── package.json
└── .gitignore
```

### Step 4: Configure TypeScript

Create `tsconfig.json`:

```json
{
  "compilerOptions": {
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [
      {
        "name": "next"
      }
    ],
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

### Step 5: Configure Next.js

Create `next.config.js`:

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {}

module.exports = nextConfig
```

### Step 6: Update package.json Scripts

Add these scripts to your `package.json`:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  }
}
```

### Step 7: Create App Router Files

**app/layout.tsx** (Root Layout):

```typescript
import type { Metadata } from "next";
import "./globals.css";

export const metadata: Metadata = {
  title: "Next.js App",
  description: "Generated by create-next-app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

**app/page.tsx** (Home Page):

```typescript
export default function Home() {
  return (
    <main>
      <h1>Welcome to Next.js!</h1>
    </main>
  );
}
```

**app/globals.css** (Global Styles):

```css
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  font-family: system-ui, -apple-system, sans-serif;
}
```

### Step 8: Run the Development Server

```bash
npm run dev
```

## Project Structure (App Router)

After setup, your Next.js project with App Router will have this structure:

```
my-nextjs-app/
├── app/                    # App Router directory
│   ├── layout.tsx         # Root layout (required)
│   ├── page.tsx           # Home page
│   ├── globals.css        # Global styles
│   ├── loading.tsx        # Loading UI (optional)
│   ├── error.tsx          # Error UI (optional)
│   └── not-found.tsx      # 404 page (optional)
├── public/                # Static assets
├── components/            # React components (optional)
├── lib/                   # Utility functions (optional)
├── next.config.js         # Next.js configuration
├── tsconfig.json          # TypeScript configuration
├── tailwind.config.ts     # Tailwind CSS config (if used)
├── postcss.config.js      # PostCSS config (if using Tailwind)
├── .eslintrc.json         # ESLint configuration
├── .gitignore             # Git ignore rules
└── package.json           # Dependencies and scripts
```

## Available Scripts

Once your project is set up, you can use these npm scripts:

- **`npm run dev`**: Starts the development server at [http://localhost:3000](http://localhost:3000)
- **`npm run build`**: Creates an optimized production build
- **`npm run start`**: Starts the production server (run after `build`)
- **`npm run lint`**: Runs ESLint to check for code issues

## Adding Tailwind CSS (Optional)

If you didn't add Tailwind during setup, you can add it later:

### Step 1: Install Dependencies

```bash
npm install -D tailwindcss postcss autoprefixer
```

### Step 2: Initialize Tailwind

```bash
npx tailwindcss init -p
```

### Step 3: Configure tailwind.config.js

```javascript
module.exports = {
  content: [
    "./pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

### Step 4: Add Tailwind Directives to CSS

In your `app/globals.css`:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

## Common Next Steps

After creating your Next.js project:

1. **Create additional pages**: Add new folders in `app/` directory
2. **Add components**: Create a `components/` folder for reusable components
3. **Set up environment variables**: Create `.env.local` for local environment variables
4. **Configure metadata**: Update `metadata` in `layout.tsx` for SEO
5. **Add API routes**: Create `app/api/` directory for API endpoints
6. **Set up database**: Integrate your preferred database (PostgreSQL, MongoDB, etc.)
7. **Add authentication**: Set up NextAuth.js or your preferred auth solution

## Troubleshooting

### Port Already in Use

If port 3000 is already in use:

```bash
npm run dev -- -p 3001
```

### TypeScript Errors

If you see TypeScript errors, make sure:
- All dependencies are installed: `npm install`
- TypeScript is properly configured in `tsconfig.json`
- You're using the correct file extensions (`.tsx` for React components)

### Build Errors

If the build fails:
- Clear the `.next` folder: `rm -rf .next` (or delete it manually)
- Reinstall dependencies: `rm -rf node_modules && npm install`
- Check for TypeScript errors: `npm run lint`

## Summary

Creating a Next.js project is straightforward:

1. **Quick Method**: Use `npx create-next-app@latest`
2. **Manual Method**: Set up files and dependencies manually
3. **Run**: Execute `npm run dev` to start development
4. **Build**: Use `npm run build` for production

Choose the method that best fits your needs. The `create-next-app` method is recommended for most users as it sets up everything automatically with best practices.
